## 今日完成工作

6 份文档的大清理和一致性修复（共约 8,500 行）。

## 关键决策与理由

### 1. 代码分离：PRD → 技术规范

**决策：** 将设计文档中的所有代码块迁移到 technical-spec.md

**为什么：**
- PRD/设计文档应该关注 *做什么* 和 *为什么*
- 技术规范关注 *怎么做*
- 更易维护：代码只在一处更新
- 非技术干系人的阅读体验更清晰

**影响：**
- design.md：~1486 → 844 行（-43%）
- design_en.md：~1560 → 950 行（-39%）

---

### 2. 仅桌面客户端策略

**决策：** 删除 Web 客户端，仅保留桌面客户端

**为什么：**
- Electron 应用已覆盖所有平台（macOS/Windows/Linux）
- Web 客户端需要额外考虑浏览器端密钥存储的安全性
- 减少约 30% 的开发范围
- 安装交易软件的用户通常更喜欢原生应用

**影响：**
- 产品定位更清晰
- 开发精力更集中
- 删除了 requirement.md 和 requirement_en.md 中的 5 处引用

---

### 3. 用户认证：两个独立流程

**决策：** 分离"注册/登录"和"登录/退出"

| 流程 | 位置 | 目的 |
|------|------|------|
| 用户注册/登录 | 首次启动向导（Step 4） | 新用户初始化 |
| 用户登录/退出 | 设置面板 | 账号切换 |

**为什么：**
- 首次启动向导用于 *初始化*（创建账号）
- 设置面板用于 *管理*（切换账号）
- 不同的 UX 上下文需要不同的 UI 模式
- 避免混淆："为什么又要注册？"

**影响：**
- 功能边界更清晰
- UX 映射更合理

---

### 4. 文档一致性标准

**决策：** 强制执行版本同步

**确立的规则：**
1. 中英文版本必须具有相同的结构
2. 章节编号必须连续（6.1 → 6.2 → 6.3，不是 6.1 → 6.2 → 6.4）
3. 所有代码引用使用统一格式

**为什么：**
- 防止语言版本之间"漂移"
- 更易于贡献者维护
- 专业外观

**影响：**
- 今天修复了 2 个主要结构不一致问题
- 为未来文档建立了模板
